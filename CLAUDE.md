## Project Structure

```
songs/       — .typ source files, one per song
pdf/         — compiled PDFs (auto-generated by CI, committed back)
images/      — cover art per song + author-photo.jpg
template.typ — shared song layout (header, footer, chord highlighting)
settings.typ — author name, photo path, album, fonts, colours
index.html   — GitHub Pages site listing all songs
publish.py   — local compile script with interactive song picker
```

Key conventions:
- Each song in `songs/` imports `template.typ` via `song-template.with(...)`
- Song parameters: `title`, `cover-image`, `capo`, `soundcloud` (optional)
- `soundcloud` renders as a clickable link in the PDF footer (orange, `#ff5500`)
- PDFs are committed to `pdf/` so they are available on GitHub Pages without a build step

## GitHub Pages & CI

- Site lives at `https://sergiorykov.github.io/songs/`
- `index.html` at repo root — dark indie aesthetic, lists songs with PDF links and SoundCloud buttons
- `.github/workflows/pages.yml` — deploys to Pages on every push to `main`; self-contained (installs typst, compiles, deploys)
- `.github/workflows/compile-pdfs.yml` — triggers only when `songs/*.typ`, `template.typ`, or `settings.typ` change; compiles PDFs and commits them back with `[skip ci]`

When adding a new song:
1. Create `songs/<Title>.typ` with `song-template.with(...)`
2. Add cover image to `images/`
3. Add a `<li>` entry to `index.html` (PDF link + SC button if available)
4. CI compiles and commits the PDF automatically

## Scripts

- All scripts are written in **Python** (not PowerShell or Bash)
- Console output must be **visually polished**: use ANSI colours, bold/dim text, clear structure — similar to Claude Code's UI style
- Interactive selectors use arrow-key navigation: `msvcrt` on Windows, `tty`/`termios` on Unix — single unified `pick_song()`, no separate fallback function
- All code, comments, variable names, and docstrings are in **English**
- Print the full shell command before running it (e.g. the full `typst compile ...` invocation)
